## Quick Start (Development without Real Datasets)

### Step 1 — Install dependencies

```bash
pip install -r requirements.txt
```

On a CPU-only machine (faster install):
```bash
pip install torch torchvision --index-url https://download.pytorch.org/whl/cpu
pip install transformers scikit-learn numpy Pillow pandas tqdm matplotlib \
    seaborn faiss-cpu einops accelerate huggingface-hub scipy PyYAML \
    tensorboard pytest
```

### Step 2 — Generate mock data and run tests

```bash
# Generate synthetic DeepFashion + Polyvore data (runs in ~30 seconds)
python scripts/generate_mock_data.py

# Verify all 22 tests pass (fully offline, no GPU needed)
pytest tests/test_all.py -v
```

### Step 3 — Smoke-test train on mock data

```bash
# Train classifier (3 epochs on tiny mock data, ~2 min on CPU)
python scripts/train_classifier.py --config configs/config_mock.yaml

# Train compatibility model (needs classifier checkpoint)
python scripts/train_compatibility.py \
    --config configs/config_mock.yaml \
    --classifier-ckpt outputs/checkpoints_mock/classifier_best.pth
```

## Full Training on Real Datasets

### Step 1 — Download DeepFashion

DeepFashion requires accepting a license agreement manually:

```bash
python scripts/download_data.py --deepfashion-instructions
```

Follow the printed instructions. After downloading, extract everything to:
```
data/deepfashion/
    img/                          # ~289K garment images
    Anno/list_category_img.txt
    Anno/list_category_cloth.txt
    Anno/list_attr_img.txt
    Anno/list_attr_cloth.txt
    Eval/list_eval_partition.txt
```

### Step 2 — Download Polyvore Outfits

```bash
# Automatically downloads from Google Drive (~4GB, nondisjoint split)
python scripts/download_data.py --polyvore

# Verify both datasets are complete
python scripts/download_data.py --verify
```

### Step 3 — Train the Classifier

```bash
python scripts/train_classifier.py --config configs/config.yaml
```

**What happens:**
- Stage 1 (epochs 0–4): Only the 4 classification heads are trained (backbone frozen). This stabilises the heads before touching the pretrained weights.
- Stage 2 (epochs 5–29): The last 4 ViT transformer blocks are unfrozen and co-trained with discriminative learning rates (`backbone_lr=1e-5` vs `head_lr=2e-4`).
- Best checkpoint saved to `outputs/checkpoints/classifier_best.pth`
- TensorBoard logs at `outputs/logs/classifier_tb/`

Expected training time: ~8–12 hours on a single A100. On a V100: ~18 hours.  
Expected metrics (DeepFashion Category+Attribute benchmark):

| Metric | Expected |
|---|---|
| Category Top-1 | ~75–82% |
| Category Top-5 | ~92–96% |
| Color F1 (macro) | ~65–72% |
| Pattern F1 (macro) | ~58–68% |
| Style F1 (macro) | ~55–65% |

To resume from a checkpoint:
```bash
python scripts/train_classifier.py --config configs/config.yaml \
    --resume outputs/checkpoints/classifier_best.pth
```

### Step 4 — Train the Compatibility Model

```bash
python scripts/train_compatibility.py \
    --config configs/config.yaml \
    --classifier-ckpt outputs/checkpoints/classifier_best.pth
```

**What happens:**
- The trained classifier backbone runs once over all Polyvore items and caches 768-dim embeddings in RAM (`EmbeddingDataset`). This avoids re-running ViT every epoch.
- The small compatibility MLP (FiLM + 2-layer MLP → 64-dim embedding) is trained with BPR loss on positive/negative item pairs sampled from outfits.
- FITB (fill-in-the-blank) accuracy and AUC are evaluated after every epoch.
- Best checkpoint saved to `outputs/checkpoints/compatibility_best.pth`

Expected training time: ~2–4 hours on GPU (embedding cache computed once in ~20 min).  
Expected metrics (Polyvore Outfits nondisjoint FITB):

| Metric | Expected |
|---|---|
| FITB Accuracy | ~55–62% (random=25%) |
| FITB AUC | ~60–68% |

### Step 5 — Evaluate

```bash
# Classifier test-set evaluation
python scripts/evaluate.py \
    --module classifier \
    --ckpt outputs/checkpoints/classifier_best.pth

# Compatibility FITB evaluation
python scripts/evaluate.py \
    --module compatibility \
    --ckpt outputs/checkpoints/compatibility_best.pth \
    --classifier-ckpt outputs/checkpoints/classifier_best.pth

# Personal wardrobe crops from Person 1's pipeline
python scripts/evaluate.py \
    --module classifier \
    --ckpt outputs/checkpoints/classifier_best.pth \
    --personal-crops data/person1_crops/
```

## API Contract for Person 3

### Classifier

```python
from classifier_service import ClassifierService

svc = ClassifierService("outputs/checkpoints/classifier_best.pth")

result = svc.predict("path/to/garment.jpg")
# {
#   "category":         "blouse",
#   "category_idx":     3,
#   "color":            "white",
#   "pattern":          "solid",
#   "style":            "casual",
#   "embedding":        [0.021, -0.004, ...],   # list[float], len=768, L2-normalised
#   "top5_categories":  [("blouse", 0.82), ("shirt", 0.11), ...]
# }

# Batch inference (efficient)
results = svc.predict_batch(["img1.jpg", "img2.jpg", ...], batch_size=32)
```

### Compatibility

```python
from compatibility_service import CompatibilityService

svc = CompatibilityService("outputs/checkpoints/compatibility_best.pth")

ranked = svc.rank_compatibility(
    query_item={"embedding": [...], "category": "tops", "item_id": "001"},
    wardrobe_items=[
        {"embedding": [...], "category": "bottoms", "item_id": "002"},
        {"embedding": [...], "category": "shoes",   "item_id": "003"},
        # ...any metadata keys are passed through unchanged
    ],
    top_k=5,   # optional
)
# [
#   {"item_id": "002", "category": "bottoms", "score": 0.87, "rank": 1, ...},
#   {"item_id": "003", "category": "shoes",   "score": 0.71, "rank": 2, ...},
# ]

# Score a single pair
score = svc.score_pair(item_a, item_b)   # float in [-1, 1]
```

Person 3's FastAPI wrappers:
```python
# POST /classify
@app.post("/classify")
async def classify(file: UploadFile):
    img = Image.open(file.file)
    return classifier_svc.predict(img)

# POST /recommend
@app.post("/recommend")
async def recommend(query: ItemSchema, wardrobe: List[ItemSchema]):
    return compat_svc.rank_compatibility(
        query.dict(), [i.dict() for i in wardrobe]
    )
```

## Free Deployment Notes

| Option | Notes |
|---|---|
| **HuggingFace Spaces** (free GPU) | Deploy with Gradio or FastAPI. ViT-B/16 fits in 2GB VRAM. |
| **ONNX export** | `torch.onnx.export(model, dummy, "classifier.onnx")` then serve with ONNX Runtime on any CPU server. Reduces memory ~40%. |
| **Google Colab** | Use for training (free T4); mount Drive to persist checkpoints. |
| **Render / Railway** | Free tier (512MB RAM) is too small for ViT inference. Use ONNX + quantisation, or upgrade to paid. |

## Config Reference

Key hyperparameters in `configs/config.yaml`:

| Key | Default | Notes |
|---|---|---|
| `classifier.backbone` | `google/vit-base-patch16-224-in21k` | HuggingFace model id |
| `classifier.num_epochs` | 30 | Total epochs |
| `classifier.backbone_unfreeze_epoch` | 5 | When Stage 2 starts |
| `classifier.head_lr` | 2e-4 | Head learning rate |
| `classifier.backbone_lr` | 1e-5 | Backbone LR (Stage 2 only) |
| `classifier.mixup_alpha` | 0.2 | Set 0 to disable MixUp |
| `compatibility.loss` | `bpr` | `"bpr"` or `"triplet"` |
| `compatibility.proj_dim` | 64 | Compatibility embedding dim |
| `compatibility.num_negatives` | 4 | Negatives per positive pair |
| `fp16` | `true` | Mixed precision (CUDA only) |
